# Purpose

Refactor existing code safely.

# Workflow

1. Analyze.
2. Explain improvements.
3. Wait for approval if architecture changes.
4. Refactor.
5. Run tests.
6. Self-review.

# Checklist

Improve:

- readability
- duplication
- naming
- modularity
- maintainability

Preserve:

- API
- behavior
- compatibility

# Output Format

Summary

Changed files

Risks

Verification

# Constraints

No functional changes unless requested.
