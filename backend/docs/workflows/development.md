# Разработка

## Цель

Безопасно реализовать изменение в слоистой архитектуре проекта.

## Порядок работы

1. Изучить затрагиваемые router, service, repository, schemas и модели.
2. Зафиксировать границы изменения и проверить обратную совместимость API.
3. Определить слой реализации:
   - Router: HTTP, `Depends`, валидация и `response_model`;
   - Service: бизнес-логика и границы транзакции;
   - Repository: запросы SQLAlchemy;
   - Schema: DTO и валидация Pydantic v2.
4. Для нового или изменённого поведения добавить или обновить тесты.
5. Реализовать минимальное изменение с типами.
6. Перед коммитом выполнить проверки из [сценария тестирования](testing.md).
7. Выполнить self-review по [сценарию ревью](code-review.md).

## Обязательные правила

- Не размещать SQL и бизнес-логику в router.
- Не выполнять `commit()` в repository; транзакцией управляет service.
- Использовать только `AsyncSession` и async-вызовы для доступа к БД.
- Не возвращать ORM-модели напрямую из API.
- Не добавлять зависимости и не изменять публичный API без отдельной задачи.
- Не включать в логи пароли, JWT и секреты.

## Проверка локально

Перед запуском Docker создать `.env.docker` из `.env.docker.example` и заменить
все значения `CHANGE_ME_*`.

### PDF-экспорт в Windows

WeasyPrint требует нативные библиотеки Pango. Установите MSYS2, затем
выполните `pacman -S mingw-w64-x86_64-pango` в его оболочке. Перед запуском
API в PowerShell выполните:

```powershell
$env:WEASYPRINT_DLL_DIRECTORIES = "C:\msys64\mingw64\bin"
uv run python -m weasyprint --info
```

Задайте эту переменную в окружении процесса API или конфигурации запуска IDE.

```powershell
docker compose up -d
uv run alembic upgrade head
uv run uvicorn app.main:app --reload
```

Для локального запуска без части внешних интеграций доступна команда:

```powershell
uv run uvicorn app.main_offline:app --reload
```

## Результат
