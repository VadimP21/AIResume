# Коммиты

## Цель

Создавать маленькие, проверяемые коммиты с сообщениями, отражающими результат изменения.

## Порядок

1. Проверить `git status` и `git diff --cached`.
2. Добавить только файлы текущей задачи; не использовать `git add .`.
3. Проверить отсутствие секретов, отладочного кода и несвязанных изменений.
4. Создать commit только по явному запросу пользователя.

## Сообщение

Использовать `<type>(<scope>): <действие>` длиной до 72 символов.

- Тип: `feat`, `fix`, `refactor`, `test`, `docs`, `chore`.
- Scope указывать, когда он помогает понять область.
- Тело добавлять только для причины, риска, миграции или breaking change; не более 2 строк.

Примеры:

```text
fix(resume): фиксировать удаление резюме
test(auth): покрыть отклонение refresh JWT
docs: описать флаг test-аутентификации
```
