# 004. Добавление секции резюме всегда завершается ошибкой

- **Серьёзность:** High
- **Статус:** Open
- **Затронутые файлы:** `app/repositories/resume.py:190-212`, `app/services/resume.py:120-153`

## Описание

Запрос позиции строится с `Resume.sections.position`. `Resume.sections` — relationship, у него нет атрибута `position`; выражение воспроизводимо вызывает `AttributeError` до выполнения SQL. Даже при корректном запросе `MAX(position)` для первой секции возвращает `NULL`, который service трактует как отсутствие резюме.

## Влияние

`POST /api/v1/resumes/{resume_id}/sections` не позволяет добавить ни первую, ни последующие секции.

## Рекомендуемое решение

Заблокировать строку `Resume` отдельным запросом `FOR UPDATE`, проверить существование резюме, затем вычислить `MAX(ResumeSection.position)` по `resume_id` и назначить `1` для пустого списка. Добавить тесты для первой секции и конкурентных запросов.
