# 007. Модель версий резюме отсутствует в миграциях и metadata

- **Серьёзность:** Medium
- **Статус:** Resolved
- **Затронутые файлы:** `app/models/resume_version.py:9-26`, `app/db/models.py:1-11`, `alembic/versions/2a4721afcc53_auth_resume.py:24-69`

## Описание

`ResumeVersion` определён в коде, но не импортируется в `app.db.models`, поэтому не попадает в `Base.metadata`. Единственная миграция не создаёт таблицу `resume_versions`.

## Влияние

При подключении `VersioningService` создание версии завершится ошибкой PostgreSQL об отсутствующей таблице; autogenerate также не обнаружит модель без её импорта.

## Рекомендуемое решение

Импортировать `ResumeVersion` в модуль регистрации моделей, создать и проверить отдельную миграцию для `resume_versions`, добавить интеграционный тест сохранения версии.
