# 005. Healthcheck неверно сообщает состояние PostgreSQL

- **Серьёзность:** Medium
- **Статус:** Open
- **Затронутые файлы:** `app/api/v1/health/router.py:15-34`

## Описание

В обработчике ошибки проверки PostgreSQL изменяется `redis_status`, а `db_status` остаётся `"ok"`. Верхнеуровневый `status` всегда возвращается как `"ok"`.

## Влияние

Оркестратор и мониторинг не обнаружат недоступность базы данных; сервис может остаться в балансировке, хотя основные операции неработоспособны.

## Рекомендуемое решение

Устанавливать `db_status = "error"` при исключении PostgreSQL и вычислять общий статус из состояний зависимостей. Для неготового сервиса вернуть подходящий HTTP-статус; добавить тесты обоих отказов.
