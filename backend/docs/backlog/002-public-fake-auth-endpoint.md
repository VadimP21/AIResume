# 002. Публичный endpoint выдаёт токены тестового пользователя

- **Серьёзность:** High
- **Статус:** Resolved
- **Затронутые файлы:** `app/core/config.py`, `app/api/v1/auth/router.py`, `app/services/auth_service.py`, `tests/api/v1/auth/test_fake_auth_route.py`, `.env.example`, `README.md`

## Описание

`POST /api/v1/auth/fake_auth` не требует аутентификации и создаёт либо использует предсказуемую учётную запись с известным паролем, после чего возвращает её access и refresh JWT.

## Влияние

Любой внешний клиент может получить аутентифицированный доступ без регистрации. В production это создаёт неконтролируемую учётную запись и канал обхода обычного процесса аутентификации.

## Рекомендуемое решение

Удалить endpoint из production-маршрутов либо включать его только в development/test по явному флагу конфигурации; исключить фиксированные учётные данные из исходного кода.

## Решение

Endpoint регистрируется только при `ENABLE_FAKE_AUTH=true` в окружениях `development` и `test`. Попытка включить его в staging или production отклоняется при загрузке конфигурации. Тестовые email и пароль вынесены в обязательные переменные `FAKE_AUTH_EMAIL` и `FAKE_AUTH_PASSWORD`; фиксированные учётные данные удалены из исходного кода.
