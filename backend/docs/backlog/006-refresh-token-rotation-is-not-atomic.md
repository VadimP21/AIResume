# 006. Ротация refresh JWT допускает повторное использование токена

- **Серьёзность:** High
- **Статус:** Open
- **Затронутые файлы:** `app/services/auth_service.py:166-214`

## Описание

Ротация выполняет отдельные операции Redis `GET`, `DELETE` и `SET`. Два параллельных запроса с одним refresh JWT могут оба пройти проверку хеша до удаления, а затем оба выпустить новую пару токенов.

## Влияние

Одноразовый refresh JWT может быть использован более одного раза, что ослабляет механизм отзыва и ротации токенов.

## Рекомендуемое решение

Выполнять проверку и удаление ключа атомарно: Lua-скриптом Redis либо транзакцией `WATCH/MULTI/EXEC`; после этого выпускать новую пару токенов. Добавить конкурентный тест, в котором успешен только один запрос.
