# 006. Ротация refresh JWT допускает повторное использование токена

- **Серьёзность:** High
- **Статус:** Resolved
- **Затронутые файлы:** `app/services/auth_service.py`, `tests/unit/services/test_refresh_token_rotation.py`, `tests/integration/services/test_refresh_token_rotation_redis.py`, `docs/workflows/testing.md`

## Описание

Ротация выполняет отдельные операции Redis `GET`, `DELETE` и `SET`. Два параллельных запроса с одним refresh JWT могут оба пройти проверку хеша до удаления, а затем оба выпустить новую пару токенов.

## Влияние

Одноразовый refresh JWT может быть использован более одного раза, что ослабляет механизм отзыва и ротации токенов.

## Рекомендуемое решение

Выполнять проверку и удаление ключа атомарно: Lua-скриптом Redis либо транзакцией `WATCH/MULTI/EXEC`; после этого выпускать новую пару токенов. Добавить конкурентный тест, в котором успешен только один запрос.

## Решение

После проверки Argon2 hash refresh-ключ удаляется Lua-скриптом только при совпадении текущего значения Redis с ранее прочитанным hash. Только запрос, успешно удаливший ключ, выпускает новую пару токенов. Добавлены конкурентные unit- и Redis integration-тесты.

## Эксплуатационный риск

Redis ACL в production должен разрешать `EVAL`. При отсутствии разрешения обновление токена завершится ошибкой, а старый refresh JWT не будет повторно использован.
